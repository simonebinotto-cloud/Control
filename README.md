# Spacecraft Attitude Control Simulation

## Overview
This repository contains a Simulink model for a **3-axis closed-loop attitude control system** of a spacecraft or satellite. The project simulates the rotational dynamics of a rigid body in orbit, tracking desired Euler angles (Roll, Pitch, and Yaw) while actively compensating for environmental disturbances. 

## Model Architecture
The Simulink model is structured with a modular approach, separating the control logic from the physical plant and environmental models:

* **Reference Input:** Generates the target attitude vector `[phi, theta, psi]` that the spacecraft needs to achieve.
* **Controller Subsystem:** A modular block that processes the attitude error and calculates the required control torques. It is designed to be easily modified (e.g., implementing PID, PD, or LQR control laws) and can accommodate control allocation matrices for specific actuators like reaction wheels or magnetorquers.
* **Plant Dynamics (`EulerAng`):** Simulates the rigid body kinematics and dynamics. It takes the total applied torques and the orbital angular velocity (`w_oi`) as inputs, and outputs the current Euler angles along with the Orbit-to-Body rotation matrix (`R_ob`).
* **Environmental Disturbances:** Uses the `R_ob` matrix fed back from the plant to dynamically calculate realistic space environment torques:
    * **Gravity Gradient:** Torques generated by the uneven gravitational pull on the spacecraft's mass distribution.
    * **Magnetic Field:** Torques resulting from the interaction between the spacecraft's residual magnetic dipole and the Earth's magnetic field.
    * **External Disturbance (`T`):** An additional input block for generic or constant disturbance torques.

## Features
* **MIMO Architecture:** Fully handles 3-dimensional vectors for multi-axis control.
* **Realistic Space Environment:** Incorporates dynamic physical disturbances rather than relying purely on static noise.
* **Highly Modular:** The encapsulation of the controller and environmental models allows for easy swapping of control algorithms or testing different orbital parameters without breaking the main loop.

## Getting Started
1. Clone the repository.
2. Open MATLAB/Simulink.
3. Run the initialization script to load the necessary workspace variables (e.g., moments of inertia, orbital parameters, controller gains).
4. Open the main `.slx` model and run the simulation.
